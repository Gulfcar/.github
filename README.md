## القواعد الأساسية

* لا يتم رفع كلمات المرور أو مفاتيح API إلى GitHub.
* لا يتم رفع ملفات قاعدة البيانات أو النسخ الاحتياطية.
* مستودعات الكود الأساسية تكون Private.
* أي تعديل يتم عبر Branch منفصل.
* المهام والأخطاء تسجل في Issues.
* قبل النشر يجب مراجعة الأمان والصلاحيات.
* يجب تحديث حالة كل مشروع عند أي تغيير مهم.

---

## أولويات العمل الحالية

1. ترتيب المستودعات ومنع رفع الملفات الحساسة.
2. مراجعة تسجيل الدخول والصلاحيات.
3. إصلاح الأخطاء الأساسية.
4. تحسين الواجهات وتجربة المستخدم.
5. توثيق خطوات التشغيل والنشر.
6. تجهيز تقارير حالة لكل نظام.
